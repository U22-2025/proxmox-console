# これがおれたちのAWS

## 概要

Proxmoxコンソールを触るためには知識が必要。
仮想マシンの作成に専門的な知識を不要にし、誰もが簡単に仮想マシンを作成できるようにする。
将来的にAWS等、Terraformが対応するホストに対応できるようにする。

## 開発体制

- 2名・制作授業期間内

## 技術構成

| レイヤー | 技術 |
|---|---|
| バックエンド | Go |
| 認証 | Ory Kratos |
| VM作成・削除・ネットワーク構成 | terraform-exec |
| VM起動・停止・操作・コンソール | go-proxmox |
| インフラ初期構築 | Terraform |
| ネットワーク制御 | nftables + VLAN |
| DB | SQLite |
| 仮想化基盤 | Proxmox VE |
| サーバー | Ubuntu VM 1台（ルーター兼サービス統合） |

## 設計方針

- Ubuntu VM 1台にルーター機能（nftables）・Goアプリ・Kratosを統合
- ユーザーごとにVLANを割り当てて完全ネットワーク隔離
- ProxmoxのAPIトークンはGoアプリのみが保持
- ユーザー↔VMIDのマッピングはGoアプリのDBで管理
- 認可チェックはGoアプリ層で完結（Proxmoxのパーミッションに依存しない）
- VNCはブラウザ→Goアプリ経由のみでアクセス可能
- DBは識別子のみ保持（VMのスペック・状態はProxmox APIから取得）

## 機能

### 認証

- Ory Kratosを使用したユーザー認証
  - メールアドレス + パスワード
- ユーザーロール管理（admin / user）
- 初回ログイン時の自動ユーザー登録 + VLAN割り当て

### ユーザー機能

- kratosのユーザーIDとアプリケーション独自のデータをDBで紐付け
- ユーザーごとのVM一覧表示
- ユーザーごとのVLAN・サブネット分離による完全ネットワーク隔離

### VM管理

- VMの作成・削除（terraform-exec）
- VMの起動・停止・再起動（go-proxmox）
- VMの設定変更（スペック変更）
- Webコンソール接続（VNCプロキシ・ブラウザ→Goアプリ経由のみ）
- cloud-initによるVM起動時のコマンド自動実行（runcmd対応）
- ユーザーごとにVLANを割り当て、VM作成時に自動適用

### ネットワーク

- ユーザーごとにVLAN ID + サブネットを自動割り当て
- Ubuntu VM（ルーター）上のnftablesでルーティング・ファイアウォール制御
- デフォルトでVLAN間通信を拒否、管理者が明示的に許可
- ファイアウォールルールの追加・削除API

### 管理者機能

- 管理者ダッシュボード（リソース使用量・ユーザー数・VM数）
- 全ユーザー管理（一覧・設定変更・削除）
- 全VM管理（一覧・操作）
- ネットワーク管理（VLAN状況・ファイアウォールルール）

## 画面構成

### 一般ユーザー

| ページ | パス | 内容 |
|---|---|---|
| ダッシュボード | `/` | 統計カード、VM一覧、リソース使用量 |
| VM一覧 | `/vms` | 全VM一覧、フィルター、一括操作 |
| VM作成 | `/vms/new` | 3ステップウィザード（リソース→情報→確認） |
| VM作成進捗 | `/vms/new/status` | Terraformログ・ステータス表示 |
| VM詳細 | `/vms/:id` | 基本情報、リソース、アクション（起動/停止/再起動/削除）、VNC |
| VNCコンソール | モーダル | VM詳細から開くnoVNCブラウザコンソール |
| ユーザー設定 | `/settings` | プロフィール、VLAN情報、VM利用状況 |

### 管理者（adminロールのみ）

| ページ | パス | 内容 |
|---|---|---|
| 管理ダッシュボード | `/admin` | 全体統計、リソース概要、アラート |
| ユーザー管理 | `/admin/users` | 全ユーザー一覧、ロール変更、削除 |
| ユーザー詳細 | `/admin/users/:id` | 個別ユーザー情報、VLAN、VM一覧 |
| 全VM管理 | `/admin/vms` | 全ユーザーのVM横断管理 |
| ネットワーク管理 | `/admin/network` | VLAN一覧、ファイアウォールルール管理 |

### ナビゲーション

```
一般ユーザー:
  Home → / | VMs → /vms | 設定 → /settings

管理者 (追加):
  ── 管理 ──
  ダッシュボード → /admin | ユーザー → /admin/users
  全VM → /admin/vms | ネットワーク → /admin/network
```

- パンくずリストで階層表示
- 管理者セクションは `role=admin` のみ表示
